基于GitHub +Hexo 的博客系统搭建，主要就是下面这四步：
Step1：前期准备git ,hexo ,note.js的安装
Step2：如何把你的博客部署到Github Pages上去？

遇到的问题有如下：

  1. 博客文件名最好不要带括号，不然嵌入路径会出错
  2. 编辑博客可以在csdn 博客里编辑好，然后下载成Markdown 文件

git 常用命令：

第一次使用Git Bash Here的话，首先需要连接你的GitHub仓库
git config --global user.name " 你的GitHub用户名 "
git config --global user.email "你的GitHub邮箱 "
mkdir ttt #新建文件夹
touch a1.php #新建文件a1.php git status #查看当前文件夹下面的文件
使用git 向远程仓库提交文件，
第一步：git clone xxxxxxxx.(仓库地址)#使用git clone 命令将远程仓库文件下载到本地
第二步：将要上传的文件放进来 #以a1.php 文件为例
第三步：git add a1.php #将a1.php添加到暂存区
第四步：git commit -m "写说明" #提交到本地仓库
第五步：git push # 提交到远程仓库

下面是Hexo 常用命令

使用hexo 搭建博客步骤：
安装好hexo 之后，在Git Bash Here 下面，按下面顺序使用命令安装博客
npm config set registry "https://registry.npm.taobao.org
hexo init blogs #初始化网站目录，目录为blogs cd blogs # 进入到blogs 目录 npm install #安装hexo 框架 hexo generate #_posts 文件夹下的 markdown 文件生成静态文件也可以简写成： hexo g hexo sever #开启服务 也可简写成 hexo s
此时在浏览器上输入 http://localhost:4000/ 就可以看到博客默认界面了
hexo sever 本地调试完成后，觉得满意的话，就可以部署到GitHub仓库上托管了
在部署GitHub之前需要先取得生成 SSH KEY，其实就是生成一个公钥和密钥，因为 GitHub 需要一个密钥才能与本地相连接。执行以下命令，并 连续按 3 次回车
ssh-keygen -t rsa -C " 邮箱地址 " #邮箱地址记得换成自己账号的xxxx@xxx.com
SSH KEY 生成之后,得到了两个文件：id_rsa 和 id_rsa.pub（默认存储路径是：C:\Users\Administrator.ssh）。打开我的电脑，在地址栏输入 C:\Users\Administrator.ssh ,然后回车，即可打开这个目录 。
将id_rsa.pub 文件，复制里面的全部内容，即复制密钥，粘贴到 GitHub 的Add SSH key里
之后下一步：修改 hexo 文件夹下的 _config.yml 全局配置文件，修改 deploy 属性代码，将本地 hexo 项目托管到 GitHub 上，如下图所示： 最好用sublime打开 ， 修改完后， 一定要另存为，编码选择 UTF-8 ,否则会出现乱码

# Deployment

## Docs: https://hexo.io/docs/deployment.html
deploy:
 type: git #部署的类型
 repository: https://github.com/jaybinkoo/jaybinkoo.github.io.git # 仓库地址
 branch: master #分支名称

在生成以及部署文章之前，需要安装一个 hexo-deployer-git 插件扩展
输入 npm install hexo-deployer-git --save，然后 回车 。
此时就可以使用命令：hexo d -g ，将 hexo 项目以及文章托管到 GitHub 上
注：：hexo generate 可缩写为 hexo g ， hexo deploy 可缩写为 hexo d
部署完成了，就可以去GitHub仓库看到有新增进来的hexo 博客文件了，并且也可以通过GitHub pages 的网址 http://用户名.github.io 访问你的博客了
注：如果比之前也用过Git Bash Here连接过其他GitHub账号的仓库，这时很可能，出现报错，说发生冲突
此时需要去项目文件夹下的 .git 文件夹 修改config 文件修改成[remote "origin"] url=https://用户名:密码@github.com/用户名/仓库名.git
就可以解决冲突问题

#注：用户部署博客项目的GitHub仓库名一定要是，user name.github.io的形式
